TUI Authoring and Personal Tools

These are not all obvious first builds, but they show where AppifyHost can make terminal-native tools feel like document apps for writers, calendar users, researchers, and file-heavy workflows.

doxx .docx, .dotx Word docs are Finder-native, but terminal-native inspection/search/export is still weirdly underrepresented. WordPeek.app, viewer-first. 5 Terminal image/proportional rendering validation.
calcure .ics, calendar/task folder, agenda package Calendar plus tasks as a small local document is a good personal-ops shape. Agenda.app package around a local calendar DB and import/export. 5 Python deps, sync/cloud/weather options.
presenterm .md, .slides.md, deck folder Markdown decks are already files; opening straight into presentation mode is a native-app move. TerminalDeck.app. 4 Image/diagram rendering through ttyd.
mdt or Glow .md, note folder Markdown viewing/editing in a TUI is a good "native shell" for technical notes. MarkdownPeek.app or note-folder package. 3-4 Maturity and differentiation from browser markdown editors.
Gorae .pdf, .epub, library package A terminal librarian for PDFs/EPUBs gives a Zotero-lite angle without a giant app. Library.app with metadata index package. 4 Metadata fetching and file moves need clear behavior.
coBib .bib, YAML bibliography, paper library package Plain-text bibliography plus TUI is a strong academic document model. Bibliography.app. 4 Python/Textual packaging and citation workflow choices.
Diagram .txt, .ascii, diagram package ASCII drawing with rendered PNG output is a quirky but clean document workflow. ASCIIDiagram.app. 4 External preview/export tooling.
Yazi / superfile Folder package, project folder, media folder Modern terminal file managers can be scoped to one Finder-opened folder. FolderScope.app or package-aware explorer. 3-4 Image previews and OS open/reveal behavior through ttyd.
taskwarrior-tui .taskrc, task database folder Taskwarrior is a file-backed power-user system; a marker package can open a focused task DB. TaskScope.app. 4 Taskwarrior version/config drift.
aerc Maildir, .mbox, patch mailbox Mail and patch review are deeply document/folder-shaped. Maildir.app or patch-review profile. 3 Accounts, auth, send-side safety.

Attractors

  • Peek apps: doxx, xleak, Glow, jless. Low write risk, high native-document clarity.
  • Library packages: Gorae, coBib, calcure. Need manifest and import/export conventions.
  • Presentation mode: presenterm. The app exists mostly to open directly into use, not edit.
  • Folder scopes: Yazi/superfile/taskwarrior-tui. Good if the Finder-opened object is a deliberate folder/package, not the whole home directory.
