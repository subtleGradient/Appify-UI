TUI atlas / document-app potential

TUI Top Picks

Highest-signal TUI wrappers are the ones where the file object already exists: data files, database files, pcaps, logs, API collections, documents, and terminal-native decks.

Atlas links: method / host patterns / data / devops / authoring / prototype backlog

First-Round Ranking

Rank Candidate Document object Why it is a best-kept secret fit Wrapper sketch Risk Evidence
1 Tabiew .csv, .parquet, .json, .sqlite, .xlsx Already powers tw.app; broad tabular file support with SQL, fuzzy search, and plotting. DataPeek.app direction: one path to tw {documentPath}. OpenSSL/Homebrew/Nix packaging details. O, L, I
2 VisiData Almost any structured local data file Terminal spreadsheet multitool with a famously jagged but powerful exploration surface. VisiData.app: universal data workbench, viewer-first default. GPLv3 bundling and optional Python dependencies per format. O, I
3 Termshark .pcap, .pcapng Wireshark-like packet analysis in a terminal: display filters, streams, conversations, remote-friendly workflows. PacketPeek.app: open saved capture with termshark -r. Requires tshark/Wireshark runtime; live capture permissions should be out of scope at first. O, I
4 lnav .log, log directories, .logbundle Auto-detects log formats, merges by timestamp, indexes, filters, pretty-prints, and exposes logs as SQLite virtual tables. LogScope.app: open file/dir; optional package for saved queries and formats. Custom format scripts and enormous dirs need guardrails. O, I
5 Posting or Slumber API collection folder, OpenAPI spec, request YAML Terminal Postman shape with local/versionable request files and import paths from OpenAPI/Postman/curl. APIWorkbench.app: package with requests/, openapi.yaml, redacted envs. Secrets, scripts, and network history need explicit policy. O, I
6 Harlequin .duckdb, .sqlite, .db, profile bundles SQL IDE for the terminal, especially good where DuckDB/SQLite file documents meet a real query editor. DuckDBWorkbench.app or a richer sibling to tw.app/litecli.app. Python/Textual/adapters and credential storage for remote databases. O, I
7 doxx .docx, .dotx Terminal-native Word viewer is exactly the kind of non-obvious document app AppifyHost can make feel native. WordPeek.app: viewer-first, no editing. Image rendering and terminal protocol support need ttyd validation. P, O, I
8 presenterm .md, .slides.md, deck folder Markdown slideshow in a TUI, with themes, code highlighting, diagrams, and export options. TerminalDeck.app: open deck and present immediately. Images, animated GIFs, and terminal graphics through ttyd. O, I
9 Sampler .sampler.yml, runbook/dashboard package YAML-defined terminal dashboards from shell commands. Great as a runbook that opens to live status. Runbook.app: package can include command YAML plus read-only safe mode. Arbitrary command execution and GPLv3. O, I
10 jjui Jujutsu repo folder or marker package Jujutsu is not universal yet, but a focused repo-as-document app would feel native fast. LazyJJ.app: marker folder sibling to LazyGit. Requires jj adoption and repo model education. O, I

Quick Matrix

Quadrant Definition Best examples First prototype bias
File viewer Open one file, inspect, maybe export. Tabiew, csvlens, jless, doxx, termshark, xleak Highest. Minimal AppKit/document-state risk.
File workbench Open one file and perform richer queries/edits. VisiData, Harlequin, rainfrog, lazysql High, but write safety matters.
Package workbench Open a bundle containing many files and app state. Posting, Slumber, Sampler, lnav query bundle High if package schema is clear.
Profile portal Open a bundle that points to remote systems. termscp, Kaskade, kubetui, mitmproxy Later. Needs Keychain and redaction first.
