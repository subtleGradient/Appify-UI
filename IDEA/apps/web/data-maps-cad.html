Web Data, Maps, and CAD

WebAssembly and WebGL make browser wrappers viable for heavy local documents: analytical data, map packages, molecular scenes, and CAD geometry.

Atlas links: method / web top picks / canvas/editors / TUI data / prototype backlog

Candidate Families

Family Candidate Document shape AppifyHost angle Risk
Browser SQL DuckDB-Wasm Data package with Parquet/CSV/JSON, SQL, and derived views. Local BI workbench without a server; pairs well with packages and drag-in data. Browser memory and large-file paging.
Interactive data grid Perspective Table snapshots, streaming sources, layout config. Web Component UI for grid/charts inside DataCanvas.app. Streaming state and save/load boundaries.
Declarative visualization Vega-Lite, Observable Plot .vl.json, chart package, Markdown dashboard. Chart documents can be files, not just dashboards trapped in SaaS. Data binding UX and export expectations.
Map documents MapLibre GL JS, PMTiles .mapbundle with style, GeoJSON, PMTiles, metadata. Offline maps, field notes, spatial datasets, and route review can be document packages. Tiles, licensing, file size, GPU.
Molecular visualization Mol* .pdb, .cif, molecular story package. Domain-specific document app for structures and annotations. Niche audience; large structure performance.
CAD kernel OpenCascade.js .step, .brep, code/geometry package. Near-native CAD operations in WebAssembly with a browser UI. WASM size, threading, LGPL/compliance review.
Code CAD JSCAD, Maker.js .jscad, JS model, SVG/DXF/STL exports. Parametric design documents are a good local package shape. Editor ergonomics and export fidelity.
Image/design editor Photopea API PSD/XCF/SVG/PDF and many raster formats. Technically an ideal WebView document editor. Proprietary embedding/license dependency; treat as inspiration, not a first build.

Document Package Ideas

Package Files Good first behavior
.datacanvas data/, queries/, views.json, charts/ Open, run saved SQL against local files, show grids/charts, export HTML/PNG/CSV.
.mapbundle style.json, layers/, tiles.pmtiles, notes.geojson Open offline map, toggle layers, inspect features, save annotations.
.cadbundle model.jscad, params.json, exports/, assets/ Edit parameters/source, preview geometry, export STL/DXF/STEP where available.
.molstory structure.cif, views.json, annotations.md Open structure, step through named views, save measurements and annotations.
