Jobs To Be Done

The product is not a generic node editor. The product is a native-feeling way to own, open, edit, inspect, copy, and evolve SDK-based web documents as local packages.

Read order: index / problem / Webapp first / layers / package contract / dependencies / SDK profiles / golf / probes

User Jobs

Job Success condition Design pressure
Open a normal package Rename chart to chart.webapp and double-click it. Accept package.json, index.html, ESM source, relative assets, and Vite-like layout.
Create a blank document New document starts with a small working package, not an empty mystery folder. Each app needs one minimal template with canonical source and data paths.
Keep the package inspectable rg, git, Finder, backup, and manual copy remain useful. Readable JSON/source files beat opaque databases and generated-only state.
Use the SDK normally Imports and concepts feel like the upstream ecosystem. The app should alias host-owned SDK packages rather than invent new wrappers first.
Ship assets with the document Copying the package preserves images, data, fonts, and previews. Use package-relative paths and a clear assets/ or public/ convention.
Preserve advanced ceilings Custom nodes, renderers, transforms, and loaders can exist when the package declares them. Do not force all documents into lowest-common-denominator JSON.
Avoid surprise execution Opening a package does not silently install packages or run lifecycle scripts. Dependency resolution and script execution need explicit modes and logs.
Decide whether to specialize Build SDK examples as .webapp packages before minting new app extensions. Dedicated apps should be evidence-backed, not naming-first.
Upgrade over time SDK and document migrations are visible and reversible where possible. Persist format versions, host versions, and migration notes inside the package.

Non-Jobs

  • Do not become npm, pnpm, Vite, VS Code, Observable, Cytoscape Desktop, or a universal workflow platform.
  • Do not guarantee arbitrary web apps work. Guarantee a clear local package contract and good failure messages.
  • Do not make the SDK app the only way to read the package. The durable files should be ordinary enough to inspect without the app.
  • Do not privilege blank-slate generic editors over specific document workflows. Generic is the hosting layer, not the user's domain.

Definition Of Done For The Generic Layer

  • An empty package scaffolds and opens.
  • A normal package with a root index.html opens.
  • A normal package with a declared appify.json entry opens.
  • Host-owned SDK imports work without package-local node_modules.
  • Missing extra dependencies fail with a clear install/cache path, not a blank WebView.
  • Save writes canonical package files, not only browser storage.
