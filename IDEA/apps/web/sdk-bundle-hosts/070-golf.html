Risk Controls

  • Default to host-owned SDK imports and no lifecycle scripts.
  • Make every cache reproducible from package files or visibly disposable.
  • Fail loudly when custom node/render code is missing instead of rendering misleading partial documents.
  • Keep the manifest small enough that a user can read it during a bug report.
  • Prototype Webapp.app with one data-oriented SDK example and one code-oriented SDK example before generalizing.
