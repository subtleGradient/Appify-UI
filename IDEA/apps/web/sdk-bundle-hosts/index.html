Generic Web SDK Bundles

Cytoscape.js, Rete, React Flow, tldraw, and D3 are not fixed document schemas. The common AppifyHost move is to make each opened package a normal web project plus a small document contract, while the app owns the SDK capsule and local runtime.

Atlas links: canvas/editors / web top picks / host patterns / app build golf / atlas index

Thesis

  • The first app should be Webapp.app: rename a normal small web package to .webapp, double-click it, and let AppifyHost open it as a local document package.
  • The first D3, Cytoscape, Rete, React Flow, and tldraw probes should be .webapp examples, not dedicated apps.
  • The high ceiling is: custom source code, custom nodes, custom data, custom assets, and optional extra dependencies remain possible without making every document carry the whole SDK runtime.
  • The source of truth should stay in the package: code, data, graph JSON, snapshots, assets, and previews. IndexedDB and build output are caches unless the package explicitly says otherwise.
  • Dedicated SDK apps should only graduate after .webapp examples prove repeated SDK-specific affordances that a generic webapp host cannot express cleanly.

Candidate App Family

App Extension Lowest useful package SDK-specific save bridge
Webapp.app .webapp Normal web project package with optional appify.json profile. Generic file bridge first; SDK save bridges are example-level adapters.
Cytoscaper.app .cytoscape Cytoscape.js JSON, style, layout, and optional app code. cy.json() import/export for graph state.
Reter.app .rete JSON-valid nodes and connections, plus a node registry when custom node classes exist. Explicit serializer because Rete nodes can be arbitrary JS objects.
React Flow Composer.app .xyflow React Flow nodes, edges, viewport, and optional custom node components. React Flow instance toObject() or controlled nodes/edges state.
TLCanvas.app .tlcanvas tldraw document snapshot, assets, session, and current TLCanvas package files. tldraw store snapshots split into document and session state.
Dee Three.app .d3 Normal D3 web package: package.json, index.html, source, data, assets. No universal D3 data model; the app code is part of the document.
