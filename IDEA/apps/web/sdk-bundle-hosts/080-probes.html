Prototype Probes

The next implementation should prove the generic runner before any app becomes a product. The falsifier is simple: if normal packages need too much Appify ceremony, the abstraction is not generic enough.

Read order: index / problem / jobs / layers / package contract / dependencies / SDK profiles / golf

Probe Order

Order Probe What it proves Stop condition
1 Dee Three.app opens a renamed Vite-like D3 package with no package-local node_modules. Normal package entry and host SDK aliasing work. Common D3 examples need Appify-specific rewrites before they render.
2 Cytoscaper.app opens and saves graph.cyjs inside .cytoscape. Data-first SDK profile works without custom app code. Saved output loses style, layout, or graph data compared with cy.json().
3 React Flow Composer.app opens a flow.xyflow.json and a package with one custom node component. Custom renderer ceiling works without abandoning normal package layout. Custom nodes require a bespoke build path per package.
4 Reter.app opens JSON-valid nodes first, then a registered class-based node. The profile can distinguish safe JSON from serializer-required objects. The host cannot produce clear errors for missing serializers.
5 TLCanvas.app migrates current TLCanvas package files toward the shared contract. The generic contract can absorb an existing app without flattening it. Existing TLCanvas semantics become less clear or less inspectable.

Fixture Packages

  • hello-chart.d3: one index.html, one src/main.js, one CSV file.
  • small-network.cytoscape: one graph.cyjs, one stylesheet, one layout config.
  • two-node-flow.xyflow: built-in nodes only, then a custom node variant.
  • json-rete.rete: plain JSON nodes and connections.
  • snapshot.tlcanvas: tldraw document/session split plus assets.

Pass Bar

  • Open within the existing AppifyHost startup timeout or a profile-specific justified timeout.
  • Render useful content before any dependency install prompt.
  • Save a no-op document without diff churn.
  • Delete the cache and reopen from package files.
  • Copy the package to another folder and reopen with the same result.
