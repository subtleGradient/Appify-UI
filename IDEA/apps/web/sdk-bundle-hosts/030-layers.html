Encapsulation Layers

The elegant generic move is layer discipline. AppifyHost handles document identity and safe local hosting; Webapp.app proves the shared web package runner; dedicated SDK apps only appear after examples prove a real specialization boundary.

Read order: index / problem / jobs / Webapp first / package contract / dependencies / SDK profiles / golf / probes

Stack

Layer Owner Responsibilities Must not own
macOS document shell AppifyHost UTI, package open/save, window title, recent documents, startup menus. SDK-specific document semantics.
Safe local server AppifyHost Start AppServer, pass APPIFY_HOST_DOCUMENT_PATH, load APPIFY_HOST_OPEN_URL, restrict navigation. User package dependency policy.
Generic webapp host Webapp.app Open .webapp, infer entry, serve package assets, expose generic file bridge, run examples. SDK-specific product identity.
SDK capsule Webapp.app profile first; dedicated app later if justified Pinned SDK packages, adapter shell, starter templates, import aliases, optional save bridge. Per-document source of truth.
Document package User package.json, appify.json, source, graph JSON, snapshots, data, assets, previews. Host runtime internals and generated cache.
Dependency cache App or user cache directory Extra dependencies keyed by lockfile, generated bundles, transforms. Canonical document data.
Browser storage WebView cache Transient UI session, performance cache, optional local recovery. The only durable document copy.

Current AppifyHost Fit

  • contentPackage already makes a package folder the document and gives the AppServer the package as its working directory.
  • APPIFY_HOST_DOCUMENT_PATH and APPIFY_HOST_WORKING_DIRECTORY already give the web runtime stable package paths.
  • APPIFY_HOST_OPEN_URL already lets the AppServer choose a safe loopback URL or package-local file URL.
  • Navigation restriction already keeps WebView browsing inside the ready URL scope, document package, or app bundle.
  • The missing shared layer is a generic web package runner: inspect package, resolve entry, resolve dependencies, expose file/save bridges, then serve.

Generic Runner Shape

  1. Resolve the opened .webapp package and detect optional SDK profile from appify.json.
  2. If the package is empty, scaffold a normal webapp template.
  3. If no manifest exists, infer entry from index.html, src/main.*, or SDK-specific default files.
  4. Create an ephemeral runtime overlay with host SDK aliases and package-relative asset mounts.
  5. Serve a host shell that imports the package entry and injects an appifyDocument bridge.
  6. On save, write SDK-specific canonical files in the package and update preview metadata.
