02.5 / Webapp first

Webapp.app First

Before minting five SDK-branded document apps, build one generic Webapp.app for .webapp packages and use SDK examples to discover whether dedicated apps are still worth carrying.

Read order: index / problem / jobs / layers / package contract / dependencies / SDK profiles / golf / probes

The Move

  • Webapp.app handles .webapp content packages.
  • A .webapp package is a normal local web project: index.html, package.json, src/, data/, public/, and optional appify.json.
  • SDK identity starts as an example or manifest profile, not as a separate macOS app.
  • Dedicated apps only graduate after examples prove repeated SDK-specific open, save, preview, template, or file-association behavior that generic Webapp.app cannot express cleanly.

Example Packages

Example Package What it tests Dedicated app question
D3 d3-bar-chart.webapp Plain web package entry, host SDK aliasing, CSV/data asset paths. Is Dee Three.app useful, or is D3 just a normal webapp example?
Cytoscape.js small-network.webapp Graph JSON loading, style/layout files, cy.json() save bridge. Does .cytoscape deserve Finder identity for graph documents?
React Flow two-node-flow.webapp React build path, built-in nodes, custom node component lookup. Does .xyflow need first-class association, or just templates?
Rete json-rete.webapp Plain JSON graph first, serializer-required nodes second. Does Rete's serializer burden justify Reter.app?
tldraw/TLCanvas snapshot-canvas.webapp Document/session snapshot, assets, migrations, package-local persistence. Does existing TLCanvas.app remain a product, or become a Webapp profile?

Graduation Test

  • Dedicated app if the SDK wants a distinct UTI, Quick Look shape, new-document template, save bridge, or import/export menu that users would naturally expect from Finder.
  • Stay as .webapp if the package is mostly source code plus assets and the SDK is just one dependency among others.
  • Stay as .webapp if the dedicated app would only rename the same runner and add brand chrome.
  • Graduate only after at least two examples for that SDK show the same missing affordance.

Resulting Order

  1. Build Webapp.app for .webapp.
  2. Build example packages for D3, Cytoscape, React Flow, Rete, and tldraw.
  3. Measure which examples need SDK-specific persistence, templates, previews, or file associations.
  4. Only then decide whether Cytoscaper.app, Reter.app, React Flow Composer.app, TLCanvas.app, or Dee Three.app should exist.
