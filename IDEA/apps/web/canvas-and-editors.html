Web Canvas and Editors

Browser-native editors are strongest when they store open, inspectable documents: JSON canvases, Markdown, HTML wikis, flow graphs, and block documents.

Atlas links: method / web top picks / SDK bundle hosts / data/maps/CAD / host patterns / prototype backlog

Candidate Families

Family Candidate Document shape AppifyHost angle Risk
Open canvas format JSON Canvas .canvas JSON with nodes and edges; package for attachments. Best "TLCanvas sibling" because the format is open and local-first. Editor UX must exceed a simple viewer.
Drawing canvas Excalidraw .excalidraw JSON plus embedded or package assets. Ideal hand-drawn diagram app if save/asset paths are native. Many users already use web/Obsidian wrappers.
Mature diagram editor diagrams.net .drawio XML. Could be a local file wrapper where the official desktop app is too heavy or not project-specific. Existing app overlap is high.
Single-file wiki TiddlyWiki A standard TiddlyWikiFolder as a .tiddlywiki package; single-file HTML only through explicit verified open/import. AppifyHost can make local TiddlyWiki folders feel like native package documents without claiming every HTML file. Autosave and merge conflict behavior.
Block editor BlockSuite, Milkdown, Lexical Markdown, block JSON, CRDT-backed document packages. Good substrate for a local Notion-like document where package owns the data. Product design burden is higher than integration burden.
Code editor Monaco, CodeMirror Plain text, project folder, structured source packages. Useful inside specialized document apps: API clients, notebooks, DSL editors, diagram editors. Do not accidentally rebuild VS Code.
Live code sandbox Sandpack, WebContainers Small web project package with files and dependencies. Powerful for runnable examples, component demos, and code lab documents. Browser support, dependency install, security of executing package scripts.
Graph/workflow editor React Flow, Rete.js, Cytoscape.js Node/edge JSON plus assets and schema. Strong base for workflow, agent, dependency, knowledge, and simulation documents. Needs a real domain; generic node editors get muddy fast.
SDK bundle host Cytoscaper, Reter, React Flow Composer, TLCanvas, Dee Three .cytoscape, .rete, .xyflow, .tlcanvas, and .d3 packages as normal web projects plus a small Appify contract. One generic AppifyHost package runner can preserve ecosystem-normal source while the app owns SDK dependencies. Dependency resolution, custom node serialization, and surprise script execution need strict policy.

Preferred Bundle Anatomy

  • document.json or the canonical upstream file at package root.
  • assets/ for images, videos, attachments, fonts, and imported files.
  • preview.png for Finder/Quick Look later.
  • schema.json or manifest.json with format version and upstream app version.
  • exports/ only for generated artifacts that can be rebuilt or intentionally shared.
