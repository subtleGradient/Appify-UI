AppifyHost Document Patterns

The current repo already demonstrates the three wrapper shapes needed for the atlas: single files, marker folders, and real content packages.

Atlas links: method / TUI top picks / web top picks / prototype backlog

Document Modes

Mode Current probe What it means Best future candidates
fileDocument tw.app, litecli.app Finder opens one existing file. The server receives the path and starts a localhost UI for it. Tabiew, VisiData, Harlequin, csvlens, termshark, doxx, lnav on one log file.
folderMarker LazyGit.app A package folder is a marker. The parent directory is the real workspace. LazyGit siblings, jjui, K9s/kubetui context markers, repo dashboards.
contentPackage TLCanvas.app The package is the document. It can hold a manifest, records, assets, snapshots, and editor state. JSON Canvas, Excalidraw bundles, map bundles, API collections, flow/workflow bundles.

Existing App Probes

tw

  • Imports delimited tables, Parquet, JSON/JSONL, Arrow, fixed-width, SQLite, and Excel-like formats.
  • Sets TW_DOCUMENT={documentPath}.
  • Starts ttyd with tw {documentPath}, using Nix packages ttyd and tabiew when available.
  • Best lesson: broad data-file support can be mostly a plist and a path-shaped shell script.

litecli

  • Imports .db, .sqlite, and .sqlite3.
  • Sets LITECLI_DOCUMENT={documentPath}.
  • Builds a read-only SQLite file URI with ?mode=ro before launching litecli through ttyd.
  • Best lesson: write safety can be encoded in the AppServer launch path.

LazyGit

  • Exports a .lazygit package UTI, but the package is just a marker.
  • Runs lazygit --path {workingDirectory}, so Finder can open a repository without making the repository itself a package.
  • Best lesson: marker packages are the right shape for repo and workspace tools.

TLCanvas

  • Exports a .tlcanvas package UTI and uses persistent WebView storage.
  • The package holds canvas.json5, records/, snapshot.png, and readme material.
  • Runs a Bun server and prints APPIFY_HOST_OPEN_URL.
  • Best lesson: web-tech apps want package documents, not just loose JSON files.

Wrapper Contract

  • The AppServer must bind a safe local or app-bundle URL and print APPIFY_HOST_OPEN_URL=... before timeout.
  • The TUI pattern should use ttyd --interface 127.0.0.1 --check-origin --once --max-clients 1 unless a candidate needs a different terminal strategy.
  • The browser pattern should serve from the app runner and keep document mutation inside the opened package path.
  • Credential-bearing candidates need Keychain integration or a profile document that deliberately excludes secrets.
  • Viewer-first apps can launch with fewer AppKit dirty-state concerns than editors.

Package Shapes Worth Standardizing

Package Contents Candidate apps Risk
.logbundle logs/, formats/, queries/, session.json lnav Format scripts can become arbitrary work unless locked down.
.apiworkbench requests/, openapi.yaml, env.example, redacted profiles Posting, Slumber, ATAC Secrets, scripts, and network history need a policy.
.mapbundle style.json, data/, tiles.pmtiles, sources.json MapLibre, PMTiles, DuckDB-Wasm Tile licensing and large-file behavior.
.flowbundle graph.json, nodes/, assets/, schema.json React Flow, Rete.js, Cytoscape.js Toolkit does not supply product semantics.
.notebookbundle notebook.py or notebook.ipynb, data/, outputs/ marimo, JupyterLite, Sandpack Execution safety and environment reproducibility.
