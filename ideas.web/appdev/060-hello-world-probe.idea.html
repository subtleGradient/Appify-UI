Hello World Appdev Probe

Before creating appify-ui.appdev/Appify UI 26.app, prove the smallest useful shape: an .appdev bundle that contains a tiny AppifyHost app and a sibling document package that opens one Quick Look HTML preview file.

Sequence: 000 capture / 030 bundle shape / 040 two proposals / 050 map reduce

JSON5 Preference

  • Prefer .json5 over .json for Appify UI manifests and examples.
  • Bun has native JSON5 support, and JSON5 is better for human-authored app recipes because comments and trailing commas are useful.
  • Keep Info.plist as plist because that is the macOS bundle contract.

Target Example

Path Role Rule
examples/hello-world.appdev/Hello.app Smallest AppifyHost app Handles .hi bundles. No Bun, no local HTTP server, no templates, no extra controls.
examples/hello-world.hi/QuickLook/Preview.html Smallest content package document When the .hi bundle is double-clicked, Hello.app opens this HTML file in a document window, and Finder Quick Look previews the same file.
examples/hello-world.appdev Workshop container For the first probe, it may contain only Hello.app. Let the absence of extra files create pressure for the next question.

Runtime Contract

  1. Hello.app declares .hi as a package document type.
  2. AppifyHost opens the .hi package as contentPackage.
  3. The app-local script prints APPIFY_HOST_OPEN_URL=file://{documentPath}/QuickLook/Preview.html.
  4. The WebView loads the file because AppifyHost permits local file URLs under the opened document package.
  5. The script stays alive so the AppifyHost document window does not immediately treat the app-local process as exited.

Questions This Should Beg

  • Does hello-world.appdev need an appdev.json5, or is a single nested app enough for the first mental model?
  • Does Hello.app need bundled AppifyHost source, or is a copied runtime binary enough for the first zero-feature example?
  • Should appdev examples keep documents inside the appdev bundle, beside it, or both?
  • Should a double-clickable example include installer/Launch Services registration instructions, or is direct open -a Hello.app hello-world.hi enough at first?
  • What is the smallest useful "hack me" affordance that does not break the zero-feature promise?
