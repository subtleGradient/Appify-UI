Boundary Rule

Question Ships in .app? Lives in .appdev?
Does it explain how the app works? Yes Optional mirror or draft
Is it needed to run, inspect, rebuild, or hack the app? Yes, if license and size allow Maybe, as source input
Does it explain why the creator chose this design? No by default Yes
Is it a prompt, scratch note, rejected path, task log, private screenshot, or credential? No Yes, if appropriate
Is it generated output? Only if required at runtime or useful for hacking Generated outputs should be declared, not silently mixed with notes

First Prototype Experiment

  1. Create experiments/0001-host-boots-own-app/.
  2. Read Appify UI 26.app/Contents/Resources/Developer/appify-app.json5.
  3. Validate only kind, schemaVersion, name, entry, and host.selfHosted.
  4. Render a host shell showing the current app identity.
  5. Prove that opening a foreign .appdev inspects manifests and files without running a nested executable.
  6. Load a tiny hello.app fixture through the same loader path.
  7. Record whether one loader can treat the host app and child apps uniformly enough that "self-hosted AppifyHost" is real.
