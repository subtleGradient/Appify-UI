Appify UI 26 Hyperslices

The idea is not a GUI around build scripts. It is a way to make the path from "I have a local tool" to "I have a double-clickable AppifyHost Mac app" feel obvious, reversible, and fun.

Binary Slices

Razor Half A Half B Implication
Source of truth Manifest-first App-bundle-first Manifest-first can generate Info.plist, server env, document types, scripts, fixtures, and smoke checks. App-bundle-first teaches by direct inspection of a working .app. Choose a dominant mode, then borrow from the other.
Creation mode Wizard Hackable exemplar Wizard lowers first-use friction. Exemplar preserves the repo's spirit: duplicate it, change anything, run it, ship your version.
Output posture Generator emits apps Workshop owns apps Generated apps stay clean and atomic. Workshop-owned apps feel richer but risk hidden state and project lock-in.
User promise Easy Fun Easy wants guardrails, defaults, validation, and previews. Fun wants tangible toys: clone this app, rename it, swap the runner, run a smoke test, see it work.
Runtime model Existing command wrapper New web runner scaffold Command wrappers align with tw, litecli, and LazyGit. Web runner scaffolds align with TLCanvas. Treat them as separate first-class paths.
Document model Known AppifyHost modes Invented app package schema The known modes are fileDocument, folderMarker, and contentPackage. New schemas should start as named skeletons, not a meta-format for everything.
Feedback loop Static recipe validation Live app launch Static checks catch missing extensions, unsafe tokens, and bad paths. Live launch is the magic: start the generated app, open a fixture, watch APPIFY_HOST_OPEN_URL, inspect logs.
Hackability surface Plain files and scripts Opaque polished UI The UI should reveal and improve the generated files, not replace them as the true artifact.
Safety posture Local/read-only by default Capability escalation Default recipes should avoid secrets and mutation. Network, credentials, shell mutation, Keychain, and write-enabled DB tools become explicit capability badges.
Recursion Meta-app makes siblings Meta-app can make itself Making siblings is safer. Making itself is the delightful recursive proof, but should be framed as the hackable implementation, not a required editing workflow.

Robust Slices

  • Manifest-first vs app-bundle-first is the central product fork.
  • Easy vs fun should not collapse into "lots of UI"; easy is fewer decisions, fun is faster visible evidence.
  • Shippable vs private is the ethical boundary: what the recipient needs to run and hack ships; personal design process stays outside.
  • Signed runtime vs hackable source is the technical boundary: hacking should usually flow through source and regeneration, not mutate a trusted app in place.

Two Natural Clusters

Cluster Primary good Core move Failure to avoid
Recipe Foundry Easy Open .appdev, edit a compact recipe, validate it, generate a root .app, and run smoke checks. A boring form generator that hides the interesting files.
Living App Dollhouse Fun Use Appify UI 26.app itself as the nested tutorial, template, and live object to duplicate and mutate. A recursive toy that becomes hard to sign, update, or reason about.
