Values

The project is grounded in durability, inspectability, bottom-up creativity, and a deep trust that ordinary web standards are finally strong enough to carry playful personal software.

Core Values

Value Meaning Design Pressure
Creative sovereignty People should make small interactive things without permission from a platform, build chain, registry, or cloud account. Default to plain files, local open, and no install step.
Durability A daughter's story, drawing, game, or form should keep working after platforms rot. Favor browser-native files, relative links, and visible assets.
Same bytes The artifact should not become different things locally, on Vercel, on GitHub Pages, in email, or on an SD card. Reject publish pipelines that rewrite the source of truth.
Inspectability Finder, git, rsync, grep, Time Machine, and human eyes remain first-class tools. No hidden database as the only durable document.
Grassroots composition People should be able to invent their own systems gradually from strict small pieces. Prefer conventions and links over central schemas.
Agent friendliness AI agents should operate on the same artifacts humans can inspect. Use semantic HTML, small files, explicit links, and low-token surfaces.
