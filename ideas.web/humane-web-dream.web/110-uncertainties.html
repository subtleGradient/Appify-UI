Uncertainties

These are the live gaps. They should stay visible so future work can probe them instead of quietly hardening assumptions.

Open Questions

  • How far can single-file webforms go before bundle-shaped forms are needed?
  • What is the smallest useful component-kit convention?
  • How should public, private, studio, tasks, and kit bundles be named without creating bureaucracy?
  • What local storage behavior belongs in Web.app versus in authored files?
  • How should generated previews be refreshed without making the package depend on automation?
  • What is the right first mobile subset for iOS and Android Web.app ports?
  • How much old-web compatibility belongs in Web.app before it becomes a server simulator?
  • How can Web.app support decentralized storage, identity, provenance, and sharing without importing wallet or token assumptions? See Portable Web Trust.

Falsifiers

Claim What Would Redraw It
Static web files can carry most early creativity. Repeated real creative probes require build tools before they become interesting.
Visible naming is enough to separate public and internal work at first. Users repeatedly publish private studio material despite clear bundle names and previews.
Web components can be the leverage layer. Useful components repeatedly require framework-level state machinery before they are reusable.
