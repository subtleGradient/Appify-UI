Constraints

Constraints are the creative affordance. The trick is to make them strict enough to unlock trust and loose enough to let people invent their own systems.

start / decisions / interactive creativity / component kits

Strict Rules

  • The simple path must open without npm, a package manager, a build step, or a dev server.
  • The same files should work locally, zipped, emailed, copied to removable media, and statically hosted.
  • State that must travel with the artifact must live in visible files inside the artifact.
  • Relative links and imports are the primary composition mechanism.
  • Unknown power must fail loud instead of becoming silent corruption or hidden state.
  • Do not execute bundle-provided server code to make old or dynamic web patterns work.

Allowed Power

Rich interaction is allowed. Games, animations, generative art, custom elements, form-associated custom elements, IndexedDB caches, service workers, drag and drop, canvas, audio, and WebGL can all belong. The boundary is not "no power." The boundary is that the durable artifact remains understandable as web files.
