Agent Collaboration

AI agents should collaborate through web-native files when the work itself is web-native. The artifact should teach the agent the terrain without forcing the human to maintain a separate markdown shadow world.

start / values / webforms / component kits / uncertainties

Agent Pit Of Success

  • Small files with clear names beat one giant manifesto.
  • Hyperlinks carry context better than hidden project memory.
  • Semantic HTML gives agents headings, tables, lists, forms, and links without custom parsing.
  • Component kits let agents generate richer interfaces with fewer tokens.
  • Visible constraints keep agent work from drifting into framework setup.

Boundary Between Public And Internal

The first answer is visible shape, not policy machinery. Use separate bundles like public.web, studio.web, tasks.web, and kit.web when that makes the work easier to see. Do not force one universal taxonomy before real projects prove where the names want to settle.
