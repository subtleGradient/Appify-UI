Method and Slices

This atlas maps AppifyHost opportunities: tools that become much more obvious when a Finder document, folder, or package opens straight into a focused TUI or browser-powered workbench.

Atlas links: host patterns / TUI top picks / TUI data / TUI devops / TUI authoring / web top picks / prototype backlog

Map Object

The object is not "cool terminal apps" in general. The object is the narrower terrain where a path-like artifact already exists and a wrapper app can make the artifact feel native: double-click, own UTI, window title, local server, direct save/export, and no cloud account required.

Probe Inventory

Existing AppifyHost Probes

  • tw.app: single file document routed to tabiew for delimited, JSON, Parquet, SQLite, Arrow, fixed-width, and Excel-like data.
  • litecli.app: single SQLite file routed to litecli in read-only URI mode.
  • LazyGit.app: folder marker package whose parent directory is the real working tree.
  • TLCanvas.app: content package hosted by a Bun/tldraw runner.

Research Probes

  • Five parallel subagents: data TUI, devops/security TUI, authoring/personal TUI, web-tech wrappers, and local codebase explorer.
  • Direct web checks for current official docs, project pages, package pages, and format references.
  • Local plist/script inspection for document-mode constraints.

Terrain Map

Home Territory

  • One document path goes into one command and yields a useful screen.
  • Viewer-first defaults; write operations are explicit and recoverable.
  • Public file formats or packages with a visible manifest.
  • No account, daemon, cloud sync, browser extension, or global OS mutation needed.

Edge Territory

  • Credential profile bundles that are safe only with Keychain integration.
  • Terminal graphics that may degrade in ttyd/xterm.js.
  • Copyleft projects where bundling/distribution needs deliberate compliance.
  • Large WASM or Python stacks where startup time could break the native illusion.

Enemy Territory

  • Cloud-only apps with no local save story.
  • Tools that require privileged daemons as the normal path.
  • Proprietary embedding without a clear license.
  • Tools whose UI depends on a desktop terminal feature that xterm.js cannot represent.

Falsifiers

  • A candidate falls below priority if its official CLI cannot open the document path without hidden global config.
  • A TUI falls below priority if ttyd cannot render its required images, mouse model, or key chords acceptably.
  • A web-tech candidate falls below priority if it cannot save back to a stable local file/package without cloud mediation.
  • A prototype should stop if the license does not allow the intended bundle/distribution model.
  • Any candidate that handles secrets must graduate only after a Keychain and redaction design exists.
