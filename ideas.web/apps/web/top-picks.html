Web Tech Top Picks

TLCanvas proves the web-tech lane: wrap a local browser app, make the package a real macOS document, and let excellent web libraries do the hard interaction work.

First-Round Ranking

1 JSON Canvas + app ecosystem .canvas, .canvasbundle Open file format for infinite canvas data. Obsidian and Charkoal store it directly; Kinopio, Flowchart Fun, hi-canvas, and OrgPad prove import/export pressure. JSONCanvas.app: render/edit .canvas, preserve unknown fields, and add optional package attachments. Spec is conservative; product value comes from editor UX and compatibility fixtures. O, I
2 Excalidraw .excalidraw, asset bundle Open JSON drawing format, familiar hand-drawn diagram feel, strong offline story. Excalidraw.app: package JSON plus embedded assets. Existing wrappers; asset path/save semantics need care. O, I
3 TiddlyWiki standard .tiddlywiki folder package; explicit import/open for verified single-file wikis later. It is already a self-contained web document. AppifyHost can solve the browser save friction. WikiDock.app: native package host for TiddlyWikiFolders without claiming generic HTML files. Single-file diffs and autosave conflict behavior. O, I
4 marimo .py notebook, notebook package Reactive notebook stored as pure Python; much better document ergonomics than opaque notebook JSON. MarimoDoc.app: open one notebook and run local server. Python env management and execution trust. O, I
5 MapLibre GL JS + PMTiles .mapbundle: style JSON, GeoJSON, tiles A map can be a local document if the package owns its style, data, and tile sources. MapBundle.app: WebGL map editor/viewer for local/offline maps. Tile licensing, large file handling, GPU behavior. O, I
6 DuckDB-Wasm + Perspective Data dashboard package Browser-side analytical SQL plus Web Component grids/charts makes a local BI document plausible. DataCanvas.app: package data, SQL, views, and chart state. Large datasets, memory, and worker startup. O, I
7 diagrams.net / draw.io .drawio Very capable browser diagram editor with a mature file format. DrawIOHost.app if a lightweight local wrapper matters. Existing desktop app is a strong competitor. O, I
8 React Flow or Rete.js .webapp examples first; possible later .xyflow, .rete Excellent node-editor substrates for workflows, simulators, dataflow, and agent graphs. Webapp.app: prove examples first, then decide whether React Flow Composer.app or Reter.app is still warranted. They are toolkits, not products; semantics must be invented. O, I
9 JupyterLite .ipynb, notebook package Jupyter running in-browser with WebAssembly can make notebooks portable without a full local server stack. LiteNotebook.app: static JupyterLab shell plus package-backed files. Feature gaps versus full Jupyter; browser filesystem bridging. O, I
10 OpenCascade.js / JSCAD .step, .brep, .jscad, CAD package WASM CAD kernels and code-CAD turn serious geometry into a local browser document. CADSketch.app: code, preview, exports, assets. WASM size, licensing, and CAD UX complexity. O, I

What Makes Web Tech Native Enough

  • The package must own the durable document model, not just browser IndexedDB.
  • WebView persistence should be a cache, not the only source of truth.
  • Assets belong beside the primary JSON/HTML/source file so Finder copy/backup works.
  • The wrapper should provide document identity, sandboxed local server, and explicit import/export.
