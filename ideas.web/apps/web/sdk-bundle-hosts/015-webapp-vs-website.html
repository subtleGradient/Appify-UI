.webapp vs .website

These bundle types are fundamentally different. .webapp is for local app packages that may use npm ecosystem tooling; .web and .website are for static web-native documents with no build-step expectations.

Read order: index / problem / jobs / Webapp first / layers / package contract / dependencies / probes

Hard Split

Bundle Meaning Allowed Not allowed
.webapp A local web application package. package.json, npm dependencies, TypeScript, JSX, Vite-like source, SDK profiles, cache-backed transforms. Silent dependency install, hidden canonical state, lifecycle scripts by default.
.web A static web document package. index.html, CSS, JavaScript modules, images, fonts, JSON, CSV, relative links, Service Worker only if already authored as static web tech. package.json, node_modules, npm install, Vite/Webpack/Rollup expectations, TypeScript compile expectations.
.website A static website package with the same constraints as .web. Multiple pages, assets, data files, static routing, prebuilt browser-native JavaScript. Any npm ecosystem dependency resolution or build step needed at open time.

Rule

  • If opening the bundle requires npm, a package manager, a bundler, TypeScript compilation, JSX transform, dependency cache, or SDK aliasing, it is .webapp.
  • If opening the bundle is just serving static browser-native files from a package root, it is .web or .website.
  • A built artifact from a .webapp can be exported as .web or .website, but the exported static bundle must not require the original build toolchain.
  • Webapp.app and a future static website host may share AppifyHost mechanics, but they should not share dependency policy or user expectations.

Examples

Package Correct type Why
D3 source with package.json and imported npm d3. .webapp It relies on npm package resolution.
D3 example exported to plain index.html plus prebuilt browser JavaScript. .website It is static web tech after export.
Hand-authored research site with HTML, CSS, images, and browser modules. .web or .website No package manager or build step is implied.
React Flow source package with JSX and custom nodes. .webapp It needs a transform/build/runtime dependency policy.
