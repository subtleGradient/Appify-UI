Per-SDK Profiles

Webapp.app can share the runner, but each SDK example still needs a small profile for entry inference, canonical files, import aliases, save bridge, and default template. Profiles graduate into dedicated apps only when they prove enough repeated product surface.

Profiles

These profiles are .webapp profiles. They do not apply to static .web or .website bundles unless the package has already been exported to browser-native files that need no npm or build step.

Profile Example package Default canonical file Entry inference Save risk Source anchor
cytoscape small-network.webapp graph.cyjs or cytoscape.json Graph JSON first; otherwise index.html or src/main.*. Medium: graph state is clear, but app-specific panels and algorithms are not. Cytoscape.js docs
rete json-rete.webapp graph.rete.json nodes/registry.* plus graph JSON, then normal web entry. High: Rete explicitly leaves import/export to the app when objects are not plain JSON. Rete import/export
xyflow two-node-flow.webapp flow.xyflow.json Flow JSON with built-in nodes; custom nodes require src/nodes.*. Medium: toObject() gets nodes, edges, viewport, but not arbitrary component code. React Flow save/restore
tlcanvas snapshot-canvas.webapp document.tldraw.json or existing TLCanvas package files Snapshot files first; otherwise TLCanvas current package layout. Medium: snapshots have migrations, but assets and custom shapes need policy. tldraw persistence
d3 d3-bar-chart.webapp No universal graph file; source plus data are canonical. index.html, src/main.*, or appify.json entry. High: D3 is a visualization library, so the package code is the document behavior. D3 getting started

Bridge API Shape

window.appifyDocument = {
  kind: "xyflow",
  packageURL: "/doc/",
  readText(path),
  writeText(path, text),
  readAsset(path),
  writeAsset(path, blob),
  markDirty(reason),
  save(snapshot),
  updatePreview(blob)
}

Bridge Rules

  • Built-in templates use the bridge directly.
  • Normal packages can ignore the bridge and still render.
  • Packages that want native Save semantics implement or call the bridge adapter.
  • The bridge writes package-relative paths only.
  • Every SDK profile defines which files count as canonical, session, cache, and preview.

.webapp Profile Failure Messages

  • small-network.webapp: graph JSON loaded, but no style/layout found; using profile defaults.
  • json-rete.webapp: graph contains non-JSON node type without a registered serializer.
  • two-node-flow.webapp: custom node type declared in JSON but no renderer found.
  • snapshot-canvas.webapp: snapshot schema migrated; package preview needs regeneration.
  • d3-bar-chart.webapp: no entry found; add index.html, src/main.js, or appify.json.
