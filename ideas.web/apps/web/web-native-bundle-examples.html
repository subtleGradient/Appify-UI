Web-Native Bundle Examples

Five non-trivial .web packages that exercise modern browser libraries without npm installs, build steps, custom protocols, or server code. The runnable bundles live outside this idea bundle in examples/web-native-bundles/. Each package persists its durable document through a strict ./file.ext localStorage key that Web.app can back with a real package file.

Shared Contract

  • Every bundle is a directory ending in .web with plain index.html, style.css, app.js, and README.md.
  • There is no package manager metadata, no build output, no server endpoint, and no dependency install step.
  • The durable state key starts with ./ and ends with a real extension so Web.app can route it to a writable file inside the opened package.
  • Opening the same files in an ordinary browser still works as a static site; persistence falls back to normal origin localStorage.
