TUI data terrain

TUI Data and Databases

Data files are the cleanest AppifyHost substrate: Finder already understands that a file is the thing, and terminal data tools already accept that path. The current repo's tw.app already proves most of this lane.

Tabiew .csv, .tsv, .parquet, .json, .jsonl, .arrow, .sqlite, .xlsx Already path-shaped and already in the repo. It turns "open a table" into a terminal spreadsheet/database browser with SQL. DataPeek.app or a more polished tw.app. 5 Dependency packaging.
VisiData CSV, JSON, TSV, XLSX/XLS, SQLite, HDF5, fixed-width, zip, ODS, databases, and more. It is less "viewer" than data cockpit: millions of rows, Python power, command logs, joins, pivots, reshape workflows. VisiData.app as a universal structured data workbench. 5 GPLv3 and per-format Python optional dependencies.
DataTUI CSV/TSV, Excel, SQLite, Parquet, JSON/JSONL. Newer Rust/Ratatui workbench with tabs, sort/filter builder, Polars SQL, JMESPath, and workspace snapshots. DataTUI.app, especially if workspace snapshots can become packages. 5 Young project maturity.
Datui Parquet, CSV, JSON/NDJSON, Avro, Arrow, ORC, Excel, S3/HTTP sources. Polars-powered terminal analytics and charts; strong for local data-engineering artifacts. ParquetPeek.app or DataLakePeek.app. 5 Very new; remote source handling needs policy.
Harlequin .duckdb, .sqlite, .db, database profile package. Terminal SQL IDE with catalog/editor/results; adapters make it a portable database cockpit. DuckDBWorkbench.app; also a richer sibling to litecli.app. 4 Python/Textual and adapter management.
rainfrog DB URL profile, SQLite/DuckDB path, Postgres/MySQL config. Lightweight database TUI with schema/table browsing, query history, favorites, and filtering. DBFrog.app for local DB and profile bundles. 4 Beta; some backends experimental.
csvlens .csv, delimiter-separated text. "less for CSV" is not broad, but it is fast, obvious, and extremely document-native. CSVPeek.app as the smallest polished proof. 4 CSV-only.
xleak .xlsx, .xlsm, .xls Fast terminal Excel viewer with formulas/search/lazy loading/export. ExcelPeek.app for forensic or lightweight inspection. 4 Viewer more than editor.
jnv / jless .json, .jsonl, .yaml, API responses. Small, sharp JSON inspectors beat generic editors for exploration. JSONPeek.app; maybe route JSONL to jnv and nested JSON to jless. 3-4 Viewer scope; jq/jaq compatibility expectations.

Home Territory

  • Data tools should default to viewer/query mode, even if upstream can edit.
  • Multi-file data packages should use a visible manifest listing source files, display names, and any derived state.
  • Remote data sources belong in a later profile-bundle tier, not the first local document tier.
  • SQLite and DuckDB are the best bridge formats because they already feel like documents.
